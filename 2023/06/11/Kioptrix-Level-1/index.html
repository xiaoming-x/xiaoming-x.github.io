From

靶机地址：Kioptrix Level 1

信息收集：

端口优先级：443=80>139>22

目录扫描

访问443与80，页面内容一致，无有用信息

目录扫描

访问test.php

Samba（解法一）

信息中给出了samba的信息

msf获取靶机信息

版本为Samba 2.2.1a

找一下对应版本的exploit

选远程命令执行Samba < 2.2.8 (Linux/BSD) - Remote Code Execution

定个位

使用gcc编译

了解下exploit的大概用法

Apache（解法二）

结合nmap扫描信息中，80、443使用了Apache/1.3.20，可找一下对应版本的exploit

可用的是这个：Apache mod_ssl < 2.8.7 OpenSSL - ‘OpenFuckV2.c’ Remote Buffer Overflow (2)

脚本复制出来再编译一手

执行脚本找一下 apache 1.3.20对应的信息

测试下来0x6b - RedHat Linux 7.2 (apache-1.3.20-16)2可以用

这里有点问题，看些有大佬连上去就是root，我连上去就只是apache，而且断了再连还上不去

部分问题

问题一：

编译时提示找不到openssl/ssl.h

fatal error: openssl/ssl.h: No such file or directory

解决办法：

安装OpenSSL开发包

参考链接：“ fatal error: openssl/ssl.h: No such file or directory “错误解决

问题二：

靶机开机无法获取IP

修改靶机的VMX中配置信息为NAT，避免启动后无法自动获取IP地址

nat 、host only 、桥接可根据实际情况配置，但默认的桥接模式貌似获取不到IP，建议nat或者仅主机
