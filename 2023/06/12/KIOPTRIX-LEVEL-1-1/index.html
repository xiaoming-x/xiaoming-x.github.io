80 HTTP Apache httpd 2.0.52

443 HTTPS Apache httpd 2.0.52

HTTP/HTTPS

个人思路：SQL注入>暴力破解>目录扫描

SQL注入

登录抓包，直接扔SQLmap（不会手工，后面补一手）

注入得到有用信息

id password username
1 5afac8d85f admin
2 66lajGGbla john

分别登录web查找有用信息点，使用john登录提示

切admin

执行命令

反弹shell

检查当前存在用户

结合之前得到的john密码，切一下john，切换失败

检查系统内核 CentOS release 4.5

找一找相关的exploit

综合信息选择linux_x86/local/9542.c

EXP：9542.c属于一个location本地执行的漏洞，所以我们需要开启web服务，将文件下载到靶机上再进行编译

具体是啥也没查到
